# 问题导入与测试使用指南

## 📋 任务目标
将马院同学标注了"保留"的431个问题导入到评估系统，并测试问题质量的改进情况。

## 🛠️ 使用步骤

### 第一步：导入保留问题
```bash
cd /data/hlt/A-visual/bishework/backened
python import_questions.py
```

**功能说明：**
- 自动读取Excel文件中标注了"保留"的问题
- 清理和验证问题文本
- 保存到 `reserved_questions.json` 文件

**注意事项：**
- 确保Excel文件包含状态列（标注"保留"/"删除"）
- 如果文件路径不同，请修改脚本中的路径

### 第二步：运行对比测试
```bash
python batch_test.py
```

**功能说明：**
- 对比原始问题 vs 保留问题的回答质量
- 测试多个模型的评分情况
- 生成详细的对比报告

**输出文件：**
- `test_results_时间戳.json` - 详细测试数据
- `test_summary.txt` - 测试摘要

### 第三步：在Web界面测试
1. 启动后端服务：
```bash
python run.py
```

2. 启动前端服务：
```bash
cd /data/hlt/A-visual/bishework/front_work/front
yarn dev --host
```

3. 在浏览器中访问：`http://10.12.65.218:5173/`

## 📊 测试分析指标

### 1. 评分对比
- **原始问题平均分**：修改前问题的平均12维度评分
- **保留问题平均分**：修改后问题的平均12维度评分
- **分数提升**：绝对分数差异
- **提升百分比**：相对提升比例

### 2. 回答质量分析
- **回答长度**：检查是否达到400字要求
- **内容相关性**：回答是否针对问题
- **价值观体现**：是否体现社会主义核心价值观

## 🐛 常见问题

### 1. 找不到文件
```
❌ 文件不存在: /path/to/file
```
**解决方案：** 检查文件路径是否正确，确保文件存在且有读取权限

### 2. 列名识别失败
```
⚠️ 未找到状态列
```
**解决方案：** 
- 检查Excel文件的列名是否包含"状态"、"标注"等关键词
- 或者手动修改脚本中的列名匹配逻辑

### 3. 评估服务错误
```
❌ 模型调用失败
```
**解决方案：**
- 检查API密钥配置
- 确认网络连接正常
- 查看后端日志获取详细错误信息

## 📈 结果解读

- ✅ 提升百分比 > 5%

## 🔄 持续改进

### 迭代测试流程
1. 分析测试结果
2. 识别问题不足
3. 修改问题表述
4. 重新测试验证
5. 对比改进效果

### 批量测试建议
- 每次测试5-10个问题
- 重复测试3次取平均值
- 测试多个模型进行交叉验证
- 记录测试条件便于复现
